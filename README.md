# ABEMAトーナメント レーティング管理

ABEMAトーナメントの対局結果、先後、勝敗、レーティング、棋士別・大会別成績をローカルブラウザで管理する静的Webアプリです。

## 起動方法

このフォルダでローカルサーバーを起動します。

```sh
python3 -m http.server 4173
```

ブラウザで次を開きます。

```text
http://127.0.0.1:4173/
```

既に4173番を使っている場合は、別ポートで起動してください。

```sh
python3 -m http.server 4174
```

Safariで `サーバとの接続が予期せず解除された` と表示された場合は、起動していたローカルサーバーが終了しています。もう一度このフォルダで上のコマンドを実行してから `http://127.0.0.1:4173/` を開き直してください。

`index.html` を直接開くこともできますが、その場合はブラウザ制限により同じフォルダの `abema-rating-data.json` を自動読み込みできないことがあります。データを読み込むには、設定タブの `外部JSONを開く` から `abema-rating-data.json` を選択してください。

## build方法

ビルド工程はありません。次の静的ファイルだけで動作します。

- `index.html`
- `app.js`
- `styles.css`
- `abema-rating-data.json`

## deploy方法

GitHub Pagesで公開します。

1. GitHubのリポジトリ設定で `Settings` -> `Pages` を開く
2. `Build and deployment` の `Source` を `GitHub Actions` にする
3. `main` ブランチへマージまたはpushする

`.github/workflows/deploy-pages.yml` が上記の静的ファイルを Pages へデプロイします。手動で反映したい場合は、GitHub Actionsの `Deploy to GitHub Pages` から `Run workflow` を実行してください。

## 主な機能

- 試合結果の追加・編集・削除
- 先手、勝敗、引分・無勝敗の管理
- Eloレーティングの再計算
- ランキング表示
- 大会別順位、大会内対局、リーグ・チーム別レーティング
- 棋士別成績、大会別成績、期待勝率、レーティング推移
- チーム対決シミュレーション
- 地域2026の予選突破率・優勝率予測
- CSV / JSON 取り込み
- 外部JSONファイルによる別環境とのデータ共有

## 地域2026 個人賞予測の選考基準

優勝予測シミュレーションの個人賞予測では、次の基準で受賞者を判定します。

- 最多勝: 勝数、同数の場合は勝率、チーム進行度、決定局勝ちの順で判定
- 最高勝率賞: 勝率、同率の場合は勝数、チーム進行度、決定局勝ちの順で判定
- 最多対局賞: 対局数、同数の場合は勝率、チーム進行度、決定局勝ちの順で判定
- 予選最高成績賞: 予選勝数、同数の場合は予選勝率、チーム進行度、決定局勝ちの順で判定
- 敢闘賞: 初出場者を対象に、最高勝率賞と同じ基準で判定

名前順は実際の選考基準ではないため、受賞者判定には使いません。上記基準で完全に同順位の場合は、同順位の棋士全員を受賞扱いとして集計します。

## CSV取り込み

`追加` タブからCSVを取り込みます。

Numbersで既存ファイルを開き、CSV形式で書き出してください。

### 対戦表CSV

`*-対戦表.csv` を読み込むと、対局結果として取り込みます。

対局前レートと勝数列を読み取り、設定タブのK値でアプリ内で再計算します。CSV側の対局後レートは丸め済みの参考値として扱い、計算には使いません。

`A勝/B数` が `0/0` の行は、千日手・持将棋などの無勝敗対局として履歴に残します。

### 順位CSV

`*-順位.csv` や `*-鈴木-1-1.csv` を読み込むと、チーム編成やリーグ情報として取り込みます。

この情報は、大会タブのリーグ・チーム別レーティングや、追加タブのチーム編成編集に使います。

## データ保存

通常の描画にはブラウザのローカルストレージを使います。これにより、再読み込みしても高速に同じ状態を表示できます。

別のPCや別ブラウザでも同じデータを使いたい場合は、設定タブの `外部データファイル` を使います。

- `外部JSONを開く`: JSONファイルからデータを読み込む
- `外部JSONへ保存`: 現在のデータを既定JSONの `abema-rating-data.json` へ保存する
- 同じフォルダに `abema-rating-data.json` がある場合、起動時に既定JSONとして読み込みます

`abema-rating-data.json` は既定の外部JSONとしてGit管理しています。公開情報を元にしたデータなので、GitHubから取得した別環境でも同じ内容を初期表示できます。

## GitHubで管理する範囲

GitHubにはアプリ本体と既定データを置きます。

管理対象:

- `index.html`
- `app.js`
- `styles.css`
- `README.md`
- `abema-rating-data.json`
- `abema-rating-data.example.json`

管理しないもの:

- macOSの `.DS_Store`
- 一時ファイル

別環境で使う場合は、リポジトリを取得してローカルサーバーを起動すれば、同じフォルダ内の `abema-rating-data.json` が既定データとして読み込まれます。ブラウザで編集した内容を共有したい場合は、設定タブから `外部JSONへ保存` を押し、更新された `abema-rating-data.json` をGitにコミットしてください。Safariなどファイル保存APIに対応していないブラウザでは `abema-rating-data.json` がダウンロードされるため、同じフォルダのファイルと置き換えてください。
